Problema & Oportunidade

Porquê Wearables? Porque o utilizador vulnerável já está no centro do problema.

A procura por Cidades Inteligentes e condução autónoma está a crescer rapidamente. No entanto, os sistemas tradicionais baseados em infraestrutura — câmaras e radares fixos — revelam-se frequentemente insuficientes devido a cobertura limitada, pontos cegos e custos elevados de implementação e manutenção.

O nosso projeto aborda este problema diretamente a partir da perspetiva do utilizador vulnerável (peão / ciclista). Aproveitando os sensores presentes nos Wearables que já usamos quotidianamente, é possível detetar gestos, postura e atenção, convertendo-os em "intenções" transmitidas em tempo real para os veículos conectados.

Pontos Cegos & Cobertura

Infraestrutura estática não cobre todos os cenários de risco urbano.

Processamento na Cloud

Dados anonimizados enviados para cloud com maior capacidade computacional para inferência dos modelos de ML.

Integração V2X

As intenções detetadas são propagadas em tempo real via ITS-G5.

Privacidade & Conformidade RGPD

Os dados recolhidos — incluindo gestos e postura captados pelos Wearables — são anonimizados na origem, antes de qualquer transmissão. Os dados anonimizados são enviados para uma infraestrutura cloud para processamento pelos modelos de ML, dado o maior poder computacional disponível face ao dispositivo móvel. Nenhum dado pessoal identificável é transmitido ou armazenado externamente. Todo o tratamento de informação está em conformidade com o Regulamento Geral sobre a Proteção de Dados (RGPD / GDPR), garantindo a privacidade e os direitos dos participantes em todas as fases do projeto.

Tarefas do Projeto

Cinco frentes de trabalho integradas para uma solução completa de ponta a ponta.

Camada de Abstração

Desenvolvimento de componentes para abstração dos dados recolhidos, tornando o sistema agnóstico a diferentes dispositivos Wearables.

Sincronização Multimodal

Alinhamento de diferentes fluxos de dados e correção de desvios de relógio para criar um contexto unificado em tempo real.

Machine Learning

Recolha de datasets e treino de modelos leves para deteção de intenções — como "levantar a mão" ou "mudança de direção".

Wearable Data Hub

App Android para descoberta via BLE/Wi-Fi, aquisição de dados dos Wearables e envio para cloud para execução dos modelos de ML.

Integração V2X

Propagação dos resultados para a infraestrutura da cidade inteligente e testes em cenário real com o veículo PIXKIT.

Roadmap do Projeto

O nosso planeamento de execução ao longo do semestre (Semanas 1 a 14).

M1  ·  Semanas 1–3

Inception

Levantamento de hardware disponível, comunicação inicial entre Wearable e telemóvel, e primeira integração com processamento na cloud.

M2  ·  Semanas 4–6

Elaboration

Desenvolvimento da aplicação móvel, streaming simultâneo de múltiplos sensores, sincronização de fluxos de dados e elaboração do relatório de Elaboration.

M3  ·  Semanas 7–9

Construction

Pipeline cloud com modelos de ML, visualização e e comunicação V2X via ATCLL, e primeiro protótipo funcional com os Smart Glasses (Omi Glasses).

M4  ·  Semanas 10–14

Transition

Feedback ao utilizador em tempo real, integração End-to-End completa, testes reais com o veículo PIXKIT no ATCLL, apresentação final e poster para DETI Students.
